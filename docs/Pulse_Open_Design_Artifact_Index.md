# Индекс артефактов Pulse Open Design

## Назначение

Этот индекс помогает быстро находить локально сгенерированные артефакты Open Design, пока `pulse-ui-ux` остаётся слоем дизайн-handoff.

Файлы Open Design в `open-design-main/.od/projects/*` — локальные рабочие артефакты. Сами по себе они не являются runtime truth: каждый принятый экран всё равно нужно сверять с реальными маршрутами, контрактами и Phase 4 drift checklist.

## Текущий визуальный baseline

- Направление: `editorial-monocle`.
- Выбранная система Open Design: `notion`.
- **Канонический акцент продукта (Flutter + admin ops shell):** тёплый bronze/sand **`#C49A6C`** (`PulseTheme.seedColor` / `PulseBrandCanonical.warmBronze`).
- **Legacy OD accent:** Notion Blue **`#0075de`** сохраняется на принятых **v1 baseline HTML** до регена; новые артефакты переводятся на bronze или явно помечают blue как secondary only.
- Тон: тёплый, спокойный, premium mobile UI с тонким мотивом временных окон Pulse.
- Жёсткие ограничения:
  - без generic swipe-app визуала;
  - без ложных обещаний realtime/offline;
  - без публичных trust badges и safety scores;
  - без технического backend/debug-словаря в пользовательском тексте.

### Две визуальные семьи (с серии 10–11.05.2026)

Mobile-онбординг + auth + profile-wizard + legal long-form переведены на две **binary-locked** визуальные семьи. Любой новый экран обязан явно принадлежать одной из них; tokens менять синхронно по всей семье через ADR.

- **`splash-DNA` (Calm Dawn Aurora + Glass)** — интерактивные онбординг-/auth-/profile-экраны (signup, login, otp, profile basics, intent, privacy starter, photos). 5-stop dawn gradient + glass-карточки + iPhone 16 Pro Max frame + brand-tile с bloom-ECG. Принятые версии этой семьи перечислены ниже в таблице артефактов с тегом `splash-DNA family`.
- **`paper-frame` (Warm Paper + Reading Surface)** — long-form legal (правила сообщества полная, политика конфиденциальности полная, согласие на обработку ПДн). Тёплый кремовый desk + paper-1 reading surface + sticky topbar + reading-progress + якорное TOC, без motion-spam.

**Полный canon:** [`Docs/Rules_and_Standards/Pulse_Visual_Design_Families`](../../../Docs/Rules_and_Standards/Pulse_Visual_Design_Families.md) — runtime-truth для дизайна (tokens, frame spec, lockstep policy, универсальные UX-паттерны, DROP-only concept guardrails, a11y baseline).

**Execution log серии 10–11.05.2026:** [`Pulse_Onboarding_Wizard_Refactor_Log_2026_05`](Pulse_Onboarding_Wizard_Refactor_Log_2026_05.md) — что переделано, версии до/после, copy decisions, ARB follow-ups, backlog.

## Правило принятия

Артефакт Open Design считается готовым к implementation handoff только после того, как:

1. маршрут и ownership состояния сверены с `pulse-mobile-app/pulse-mobile-app/lib/app/routing/app_router.dart`;
2. текст не повышает уровень обещаний выше текущего runtime;
3. safety/privacy ограничения из governance-документов соблюдены;
4. визуальный язык совпадает с принятым baseline Pulse;
5. этот индекс и релевантный handoff-документ обновлены.

## Заметки по принятому Candidate Card

Сохранить:

- маршрут `/matches/slots/:slotId`;
- fullscreen above shell, без tab bar;
- направление candidate media `4:5`;
- русские labels: `Пропустить`, `Нравится`, `Безопасность`;
- мягкий progress strip без процентов;
- без точного адреса и контактов до mutual interest;
- состояния loading/empty/error.

Принято:

- пользовательские `слот` labels заменены на `окно`, где route vocabulary не нужен;
- empty/error/loading states избегают технического backend-словаря и дают понятный следующий шаг;
- route/dev hint `/matches/slots/:slotId` остаётся без изменений.
